# 🖥 Array (배열)

배열(Array)는 **동일한 자료형의 데이터**를 연속된 메모리 공간에 저장하는 자료구조이다.

## 특징
- **선형 자료구조 (Linear Data Structure)**
- **연속된 메모리 공간**에 데이터를 저장한다.
- 인덱스를 이용한 **Random Access**가 가능하다.
- 데이터 조회 속도가 빠르다.
- 데이터 삽입/삭제 시 **기존 데이터를 이동(Shift)** 해야 할 수 있다.
- 구조가 단순하여 구현이 쉽다.
---
## Array 구현
Python에서는 일반적으로 **list**를 배열처럼 사용한다.

```python
# 배열 생성
arr = [1, 2, 3, 4, 5]
print("초기 배열:", arr)

# 데이터 조회 (Access)
print("첫 번째 요소:", arr[0])
print("세 번째 요소:", arr[2])

# 데이터 수정 (Update)
arr[1] = 10
print("수정 후 배열:", arr)

# 데이터 삽입 (Insert)
arr.insert(2, 20)
print("삽입 후 배열:", arr)

# 데이터 삭제 (Delete)
arr.pop(3)
print("삭제 후 배열:", arr)
```
---
## Array 시간 복잡도

| 연산 | 시간 복잡도 |
|----------|---------|
| 접근 | O(1) |
| 탐색 | O(n) |
| 삽입 | O(n) |
| 삭제 | O(n) |

- 배열은 **인덱스를 통해 바로 접근**하기 때문에 조회는 O(1)이다.
- 삽입/삭제 시 **데이터를 뒤로 이동(shift)** 해야하므로 O(n)이다.
---
## 💡 핵심 정리

- 배열은 **연속된 메모리 공간을 사용하는 자료구조**
- **Random Access 가능 → 조회 O(1)**
- **삽입 / 삭제 → O(n)**

# 📝 면접 예상 질문
<details>
<summary>Q. 배열(Array)이란 무엇인가요?</summary>

배열은 **동일한 자료형의 데이터를 연속된 메모리 공간에 저장하는 자료구조**입니다.  
각 요소는 인덱스(index)를 통해 접근할 수 있으며, 인덱스를 이용한 접근이 가능하기 때문에 **데이터 조회가 O(1)** 의 시간 복잡도를 가집니다.

</details>

<details>
<summary>Q. 배열이 Random Access가 가능한 이유는 무엇인가요?</summary>

배열은 메모리 공간에 **연속적으로 저장(contiguous memory)** 되기 때문에  
배열의 시작 주소(base address)에 인덱스를 이용한 오프셋을 더해 **즉시 원하는 위치의 데이터에 접근**할 수 있습니다.
즉, address = base_address + (index × data_size)
공식을 통해 바로 접근이 가능하므로 배열의 조회 연산은 **O(1)** 입니다.

</details>

<details>
<summary>Q. 배열에서 삽입과 삭제 연산의 시간 복잡도가 O(n)인 이유는 무엇인가요?</summary>

배열은 연속된 메모리 공간에 데이터를 저장하기 때문에  
중간에 데이터를 삽입하거나 삭제할 경우 **뒤에 있는 데이터를 한 칸씩 이동(shift)** 해야 합니다.
</details>

<details>
<summary>Q. 배열의 장점과 단점은 무엇인가요?</summary>

#### 장점

- 인덱스를 이용한 **빠른 데이터 접근(O(1))**
- 구조가 단순하여 구현이 쉽다
- 메모리 접근 효율이 높다 (cache friendly)

#### 단점

- 중간 삽입/삭제 시 **데이터 이동이 필요(O(n))**
- 배열 크기가 고정되는 경우 메모리 낭비가 발생할 수 있다

</details>

<details>
<summary>Q. 배열(Array)과 Python의 List는 같은 것인가요?</summary>

Python의 list는 일반적인 정적 배열(static array)과는 다르게  
**동적 배열(dynamic array)** 로 구현되어 있습니다.

즉, 내부적으로 배열을 사용하지만  
필요할 때 **배열 크기를 자동으로 확장(resize)** 하는 방식으로 동작합니다.

그래서 `append()` 연산은 **평균적으로 O(1)** 의 시간 복잡도를 가집니다.

</details>
